# OpenSourceProject
The README file is usually the first file contributors to your project will read. It contains the information you want people to know right away. To update the README file, complete the following steps:

In the list of repository files, click the README.md file.

Click the Edit this file pencil icon.

Add the following information to the file:

A description of the project
An explanation of the project architecture (you can come back later and edit this information)
When you have finished your updates, scroll to the bottom of the page and click Commit changes. (Note that if the repository was protected, you would need to create a pull request at this point to ask the project administrator to pull in your changes.)

You can continue to update the README file as you develop your project. You can find some ideas for useful README content from the following resources:
